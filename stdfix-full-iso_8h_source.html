2386 // Description
2387 //
2388 // The strtofxhr, strtofxr, strtofxlr, strtofxhk, strtofxk, strtofxlk,
2389 // strtofxuhr, strtofxur, strtofxulr, strtofxuhk, strtofxuk and strtofxulk
2390 // functions convert the initial portion of the string pointed to by nptr to
2391 // short fract, fract, long fract, short accum, accum, long accum, unsigned
2392 // short fract, unsigned fract, unsigned long fract, unsigned short accum,
2393 // unsigned accum, and unsigned long accum representation, respectively.
2394 // First,they decompose the input string into three parts: an initial,
2395 // possibly empty, sequence of white-space characters (as specified by the is
2396 // space function), a subject sequence resembling a fixed-point constant; and
2397 // a final string of one or more unrecognized characters, including the
2398 // terminating null character of the input string. Then, they attempt to
2399 // convert the subject sequence to a fixed-point number, and return the
2400 // result.
2401 //
2402 // The expected form of the subject sequence is an optional plus or minus
2403 // sign, then one of the following:
2404 //
2405 // - a nonempty sequence of decimal digits optionally containing a
2406 // decimal-point character, then an optional exponent part as defined in
2407 // 6.4.4.3;
2408 //
2409 // - a 0x or 0X, then a nonempty sequence of hexadecimal digits optionally
2410 // containing a decimal- point character, then an optional binary exponent
2411 // part as defined in 6.4.4.3.
2412 //
2413 // The subject sequence is defined as the longest initial subsequence of the
2414 // input string, starting with the first non-white-space character, that is of
2415 // the expected form. The subject sequence contains no characters if the input
2416 // string is not of the expected form.
2417 //
2418 // If the subject sequence has the expected form for a fixed-point number, the
2419 // sequence of characters starting with the first digit or the decimal-point
2420 // character (whichever occurs first) is interpreted as a fixed-point constant
2421 // according to the rules of 6.4.4.3, except that the decimal-point character
2422 // is used in place of a period, and that if neither an exponent part nor a
2423 // decimal-point character appears in a decimal fixed-point number, or if a
2424 // binary exponent part does not appear in a hexadecimal fixed- point number,
2425 // an exponent part of the appropriate type with value zero is assumed to
2426 // follow the last digit in the string. If the subject sequence begins with a
2427 // minus sign, the sequence is interpreted as negated. A pointer to the final
2428 // string is stored in the object pointed to by endptr, provided that endptr
2429 // is not a null pointer.
2430 //
2431 // The value resulting from the conversion is rounded as necessary in an
2432 // implementation-defined manner.
2433 //
2434 // In other than the "C" locale, additional locale-specific subject sequence
2435 // forms may be accepted.
2436 //
2437 // If the subject sequence is empty or does not have the expected form, no
2438 // conversion is performed; the value of nptr is stored in the object pointed
2439 // to by endptr, provided that endptr is not a null pointer.
2440 //
2441 // Returns
2442 //
2443 // The functions return the converted value, if any. If no conversion could be
2444 // performed, zero is returned. If the correct value is outside the range of
2445 // representable values, a saturated result is returned (according to the
2446 // return type and sign of the value), and the value of the macro ERANGE
2447 // is stored in errno.
